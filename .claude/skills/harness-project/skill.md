# Harness — 도메인 하네스 구성 메타 스킬

사용자가 제시한 도메인에 맞는 에이전트 팀과 스킬 체계를 자동으로 설계·생성하는 스킬.

## 전제 조건

- 하네스 구성 파일은 `.claude/` 하위에 생성한다
- 에이전트: `.claude/agents/{name}.md`
- 스킬: `.claude/skills/{name}/skill.md`

## 워크플로우

### Phase 1: 요구사항 수집

사용자에게 아래 정보를 확인한다 (이미 제공된 항목은 건너뛴다):

1. **도메인**: 어떤 분야의 하네스인가? (예: 블로그, 코드리뷰, 번역, 마케팅)
2. **목표**: 최종 산출물은 무엇인가? (예: 블로그 포스트, 리뷰 리포트)
3. **에이전트 구성**: 어떤 역할이 필요한가? (사용자가 모르면 도메인에 맞게 제안)
4. **워크플로우**: 에이전트 간 협업 흐름 (순차/병렬/조건부)

### Phase 2: 하네스 설계

수집된 정보를 바탕으로 아래 설계를 작성하여 사용자에게 확인받는다:

```markdown
## 하네스 설계: {도메인}

### 팀 구성
| 에이전트 | 역할 | 사용 스킬 |
|---------|------|----------|
| {name} | {role} | {skill-name} |

### 워크플로우
Phase 1: {에이전트} → {작업} ({스킬})
Phase 2: {에이전트} → {작업} ({스킬})
...

### 산출물
- {경로} — {설명}
```

사용자가 승인하면 Phase 3으로 진행한다.

### Phase 3: 파일 생성

아래 순서로 파일을 생성한다. **모든 파일은 한국어로 작성한다.**

#### 3-1. 에이전트 파일 생성

각 에이전트마다 `.claude/agents/{name}.md` 파일을 생성한다.

**에이전트 파일 템플릿:**

```markdown
---
name: {에이전트명}
description: "{역할 설명}. '{트리거 문구1}', '{트리거 문구2}' 요청 시 사용."
---

# {에이전트명} — {역할 한 줄 소개}

당신은 {역할 상세 설명}입니다.

## 핵심 역할
1. {주요 책임 1}
2. {주요 책임 2}

## 작업 원칙
- {원칙 1}
- {원칙 2}
- {원칙 3}

## 출력 형식
\`\`\`markdown
{에이전트의 출력 형식 정의}
\`\`\`

## 협업
- {다른 에이전트와의 관계 설명}
```

#### 3-2. 스킬 파일 생성

각 에이전트의 작업 절차 스킬을 `.claude/skills/{skill-name}/skill.md`로 생성한다.

**스킬 파일 템플릿:**

```markdown
---
name: {스킬명}
description: "{스킬 설명}. {에이전트명} 에이전트가 사용하는 스킬."
---

# {스킬명} — {스킬 한 줄 소개}

## 워크플로우

### Step 1: {단계명}
- {구체적 작업 내용}

### Step 2: {단계명}
- {구체적 작업 내용}

### Step N: {단계명}
- {구체적 작업 내용}
```

#### 3-3. 오케스트레이터 스킬 생성

팀 전체를 조율하는 오케스트레이터 스킬을 `.claude/skills/{domain}-team/skill.md`로 생성한다.

**오케스트레이터 스킬 템플릿:**

```markdown
---
name: {domain}-team
description: "{도메인} 팀 오케스트레이터. '{트리거 문구}' 요청 시 사용."
---

# {Domain} Team — 오케스트레이터

{도메인} 팀을 조율하는 통합 스킬.

## 팀 구성

| 에이전트 | 역할 | 사용 스킬 |
|---------|------|----------|
| {name} | {role} | {skill} |

## 워크플로우

\`\`\`
Phase 1: {에이전트} → {작업} ({스킬})
Phase 2: {에이전트} → {작업} ({스킬})
Phase N: (조건부) {에이전트} → {작업}
\`\`\`

## 실행 방법

### Phase 1 — {단계명}
Agent 도구로 `{에이전트}` 에이전트를 호출한다:
- subagent_type: `{에이전트 타입}`
- prompt에 {필요 정보}와 {스킬} 스킬 절차를 포함

### Phase 2 — {단계명}
Agent 도구로 `{에이전트}` 에이전트를 호출한다:
- subagent_type: `{에이전트 타입}`
- prompt에 {필요 정보}와 {스킬} 스킬 절차를 포함

### Phase N — {단계명} (조건부)
{조건}인 경우:
- {에이전트}를 다시 호출하여 {작업}
- 이후 Phase {M}을 한 번 더 수행 (최대 1회)

## 최종 산출물
- `{output_path}` — {산출물 설명}
```

### Phase 4: 검증

생성된 파일을 확인하고 사용자에게 결과를 보고한다:

```markdown
## 하네스 생성 완료

### 생성된 파일
- `.claude/agents/{name}.md` — {역할}
- `.claude/skills/{skill}/skill.md` — {설명}

### 사용법
`/{orchestrator-skill-name}` 으로 팀을 실행할 수 있습니다.
```

## 주의사항

- 기존에 동일한 이름의 에이전트/스킬이 있으면 덮어쓰기 전에 사용자에게 확인한다
- 에이전트의 subagent_type은 기본적으로 에이전트 정의 파일의 name과 매칭되는 커스텀 에이전트를 사용한다
- 오케스트레이터에서 Agent 도구 호출 시 team_name을 사용하여 에이전트를 지정한다
