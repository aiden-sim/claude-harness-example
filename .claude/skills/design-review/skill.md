# Design Review — 기획서·API 설계 교차 검토

## 워크플로우

### Step 1: 산출물 수집
- `docs/prd.md` (상품 기획서) 읽기
- `docs/api-spec.md` (API 설계서) 읽기

### Step 2: 기능 매핑 검증
- PRD의 모든 기능 항목을 목록화
- 각 기능이 API 엔드포인트에 매핑되었는지 체크리스트 작성
- 누락된 API 식별

### Step 3: 과금 모델 ↔ 데이터 모델 정합성
- 과금 체계(CPC, 예산, 입찰)가 데이터 모델에 올바르게 반영되었는지 검증
- 입찰 로직과 광고 순위 공식의 기술적 구현 가능성 평가

### Step 4: 광고 정책 구현 가능성 검토
- 품질지수 계산에 필요한 데이터가 모델에 포함되어 있는지 확인
- 광고 심사 프로세스가 API 흐름에 반영되어 있는지 확인

### Step 5: 엣지 케이스 시뮬레이션
- 예산 소진 시나리오
- 동일 입찰가 경쟁 시나리오
- 광고 심사 반려 후 재등록 시나리오
- 캠페인 일시정지/재개 시나리오

### Step 6: 이슈 분류 및 리포트 작성
- 발견된 이슈를 심각도(Critical/Major/Minor)로 분류
- 각 이슈에 대한 구체적 수정 방향 제시
- `docs/review-report.md` 파일로 검토 리포트 작성
- 출력 형식은 reviewer 에이전트의 출력 형식을 따른다
